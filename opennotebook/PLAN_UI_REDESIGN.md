# UI/UX 재설계 계획
## Apple HIG + YC 스타트업 스타일

---

## 현재 상태 분석

### 구조
- 2-Column Layout: 사이드바(380px) + 메인 채팅
- 풀스크린 경전 라이브러리 오버레이
- 방법론 슬라이드인 패널

### 잘 된 점
- CSS 변수 시스템 구축됨
- Glass morphism 기본 적용
- Dark mode 기반

### 개선 필요
- 가치 제안(Value Proposition) 불명확
- 첫 화면에서 무엇을 할 수 있는지 모호
- 모바일 UX 미흡
- 마이크로 인터랙션 부족

---

## 재설계 원칙

### Apple HIG 원칙
1. **Clarity** - 모든 요소가 명확한 목적
2. **Deference** - 콘텐츠가 주인공
3. **Depth** - 레이어로 계층 표현

### YC 스타트업 원칙
1. **10초 규칙** - 10초 내 가치 이해
2. **마찰 최소화** - 클릭 수 최소화
3. **즉각적 피드백** - 모든 액션에 반응


### 1.2 Onboarding Quick Actions

## Phase 2: 레이아웃 재구성

### 2.1 Desktop Layout 개선
```
현재:
[사이드바 380px] [메인 채팅 영역]

제안 A - 중앙 집중형:
[      중앙 채팅 + 우측 컨텍스트 패널      ]

제안 B - 현재 유지 + 개선:
[사이드바 320px] [확장된 메인 영역]
```

### 2.2 반응형 Breakpoints
- Desktop: 1200px+
- Tablet: 768px - 1199px
- Mobile: < 768px

## Phase 3: 컴포넌트 개선

### 3.1 채팅 입력 영역
- 더 크고 눈에 띄는 입력창
- 플로팅 스타일 (하단 고정)
- 전송 버튼 애니메이션
- 입력 중 상태 표시

### 3.2 메시지 버블
- 부드러운 등장 애니메이션
- 출처 인라인 표시
- 복사/공유 버튼 호버 시 표시

### 3.3 경전 카드 (라이브러리)
- 호버 시 미리보기 확장
- 무한 스크롤 개선
- 필터 태그 스타일 개선

---

## Phase 4: 마이크로 인터랙션

### 4.1 로딩 상태
- Skeleton loading 전체 적용
- Shimmer 효과
- 진행률 표시

### 4.2 트랜지션
- Page transitions: 300ms ease-out
- Modal: scale + fade
- List items: staggered animation

### 4.3 피드백
- 버튼 클릭: ripple effect
- 성공: 녹색 체크 애니메이션
- 에러: shake + 붉은 테두리

## Phase 5: 타이포그래피 & 컬러

### 5.1 폰트 스케일
```css
--text-hero: 2.5rem;     /* Hero 제목 */
--text-h1: 1.75rem;      /* 섹션 제목 */
--text-h2: 1.25rem;      /* 카드 제목 */
--text-body: 1rem;       /* 본문 */
--text-small: 0.875rem;  /* 보조 텍스트 */
--text-tiny: 0.75rem;    /* 레이블 */
```

### 5.2 컬러 시스템 정제
```css
/* Primary Actions */
--primary: #a794e0;
--primary-hover: #b8a8f0;

/* Status Colors */
--success: #9bebb4;
--warning: #f5c542;
--error: #ff6b6b;

/* Neutral Scale */
--gray-100: rgba(255,255,255,0.95);
--gray-200: rgba(255,255,255,0.8);
--gray-300: rgba(255,255,255,0.6);
--gray-400: rgba(255,255,255,0.4);
```

## 최종 구현 계획

### Step 1: 풀스크린 Hero + 중앙 집중 레이아웃 (test_frontend.html)
- 사이드바 제거, 중앙 채팅 레이아웃
- 첫 화면 풀스크린 Hero 섹션
- 스크롤/클릭 시 채팅 화면으로 전환

### Step 2: 샘플 질문 버튼 추가
- Hero에 3-4개 샘플 질문 버튼
- 클릭 시 자동으로 질문 입력 및 전송

### Step 3: 채팅 UI 개선
- 중앙 정렬 채팅 영역 (max-width: 800px)
- 플로팅 입력창 (하단 고정)
- 메시지 등장 애니메이션

### Step 4: 모바일 하단 탭 바
- 3개 탭: 채팅 / 라이브러리 / 방법론
- iOS 스타일 블러 배경
- 아이콘 + 레이블

### Step 5: 경전 라이브러리 개선
- 탭으로 접근 가능하도록 변경
- 카드 hover 효과 강화
- 검색 UX 개선

### Step 6: 마이크로 인터랙션 & 폴리시
- 모든 전환 애니메이션
- 로딩 상태 개선
- 최종 반응형 테스트
